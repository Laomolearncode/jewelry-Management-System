# 饰品进销存系统领域模型与业务规则

## 1. 核心业务对象

### 1.1 系统管理
- `User`：系统操作用户。
- `Role`：角色，如系统管理员、采购员、销售员、仓管员。
- `Permission`：接口级权限点。
- `OperateLog`：操作审计日志。
- `UserToken`：登录令牌，支持接口鉴权。

### 1.2 主数据
- `Material`：材质字典，如黄金、白银、珍珠。
- `ProductCategory`：饰品分类，如戒指、项链、耳饰。
- `Supplier`：供应商档案。
- `Customer`：客户档案。
- `Warehouse`：仓库档案。
- `Product`：饰品档案，包含编码、材质、克重、证书号、价格等字段。
- `Stock`：当前库存快照，按仓库、商品、批次号、证书号维度维护。

### 1.3 业务单据
- `PurchaseOrder` / `PurchaseOrderItem`：采购单及明细。
- `SaleOrder` / `SaleOrderItem`：销售单及明细。
- `StockCheckOrder` / `StockCheckItem`：盘点单及明细。
- `InventoryTxn`：库存流水，记录所有库存变化。
- `TraceLog`：溯源日志，保留关键业务事件轨迹。

## 2. 单据状态设计

### 2.1 采购单状态
- `DRAFT`：草稿，允许修改明细。
- `APPROVED`：审核通过，允许执行入库。
- `STOCKED`：已入库，不可重复入库。
- `CANCELLED`：已作废。

状态流转：
- `DRAFT -> APPROVED`
- `DRAFT -> CANCELLED`
- `APPROVED -> STOCKED`

### 2.2 销售单状态
- `DRAFT`：草稿。
- `APPROVED`：审核并完成出库。
- `CANCELLED`：已作废。

状态流转：
- `DRAFT -> APPROVED`
- `DRAFT -> CANCELLED`

### 2.3 盘点单状态
- `DRAFT`：草稿。
- `APPROVED`：已审核并完成库存调整。
- `CANCELLED`：已作废。

状态流转：
- `DRAFT -> APPROVED`
- `DRAFT -> CANCELLED`

## 3. 关键业务规则

### 3.1 饰品档案规则
- 商品编码全局唯一。
- 证书号在有值时必须唯一。
- 克重、成本价、销售价不能为负数。
- 商品必须绑定分类、材质。

### 3.2 采购规则
- 采购单明细数量、单价、克重必须大于零。
- 采购单只有在 `APPROVED` 状态下才能入库。
- 每张采购单只允许执行一次完整入库。
- 入库时必须生成库存流水和溯源日志。

### 3.3 销售规则
- 销售单审核通过时立即出库。
- 出库必须校验指定仓库、批次号、证书号对应库存是否足够。
- 销售扣减、库存流水、溯源日志必须在同一事务内提交。

### 3.4 库存规则
- 任何库存变化都必须写入 `InventoryTxn`。
- `Stock` 只保存当前余额，不作为唯一事实来源。
- 库存维度：`warehouseId + productId + batchNo + certificateNo`。
- 禁止出现负库存。

### 3.5 盘点规则
- 盘点审核时按“实盘数量 - 账面数量”生成调整流水。
- 盈亏数量可以为正数或负数，但审核后必须同步刷新库存快照。

### 3.6 溯源规则
- 溯源支持按证书号、批次号、商品编码查询。
- 关键事件至少包括：采购创建、采购审核、采购入库、销售出库、盘点调整。

## 4. 权限模型
- 鉴权采用 Bearer Token。
- 接口级权限点格式：`模块:资源:动作`。
- 默认角色建议：
- `ADMIN`：全部权限。
- `PURCHASER`：采购与主数据只读/维护权限。
- `WAREHOUSE_KEEPER`：入库、库存、盘点权限。
- `SALESMAN`：销售、客户、库存查询权限。

## 5. 统计口径
- 库存金额：`quantity * avgCostPrice`。
- 销售金额：销售单明细总额汇总。
- 利润估算：`销售金额 - 销售成本`。
- ABC 分类：按库存金额降序累计占比，A 类 70%，B 类 20%，C 类 10%。
- 库存周转率：`期间销售成本 / 平均库存成本`。
